# IT Support Quick Guide

This guide provides beginner-friendly troubleshooting steps and best practices for common workplace IT issues.

## 1. Network Troubleshooting Steps

### Problem: No Internet Connection

1. Confirm the network cable is connected or Wi-Fi is enabled.
2. Check if other websites or services are unavailable.
3. Restart the router/modem if accessible.
4. Run a network diagnostic:
   - **Windows:** `ipconfig /all`, `ping 8.8.8.8`
   - **macOS/Linux:** `ifconfig` or `ip a`, `ping 8.8.8.8`
5. If ping to IP works but websites fail, check DNS settings.
6. Escalate to ISP or network administrator if issue persists.

### Problem: Slow Network Performance

1. Check if many users/devices are consuming bandwidth.
2. Test speed with a trusted speed test tool.
3. Confirm no large background downloads are running.
4. Restart networking equipment during maintenance windows.
5. Update router firmware if required.

## 2. System Error Troubleshooting Steps

### Problem: Computer Running Slowly

1. Restart the system.
2. Close unnecessary startup apps.
3. Check disk space and remove unused files.
4. Run antivirus and malware scans.
5. Install pending OS and software updates.

### Problem: Application Crashes Frequently

1. Record the exact error message.
2. Update or reinstall the application.
3. Verify system requirements and dependencies.
4. Check logs (Event Viewer on Windows or system logs on macOS/Linux).
5. Escalate with error details and reproduction steps.

## 3. IT Support Best Practices

- Keep systems and software patched regularly.
- Use strong passwords and enable multi-factor authentication.
- Maintain frequent backups and test recovery procedures.
- Document recurring issues and resolutions in a shared knowledge base.
- Use ticketing systems to track incidents and response times.
- Communicate clearly with users during outages and maintenance.

## 4. Escalation Checklist

Before escalating an issue, ensure the following details are included:

- User name and department
- Device name and operating system
- Description of the issue
- Time the issue started
- Troubleshooting steps already attempted
- Screenshots or error codes (if available)

Consistent documentation reduces resolution time and helps senior teams diagnose root causes faster.
